üîí Storage & Data Handling

  • Use IndexedDB via libraries like idb, dexie, or custom wrappers.
    • Wrap operations in Promise-based modules.
    • Enforce versioned schemas and handle migrations.
    • Create React hooks such as usePersistentStore() for encapsulation.
  • Provide a 2-step factory reset mechanism:
    1. Click on Reset Mechanism.
    2. Display confirmation modal.
    3. On confirm, clear IndexedDB and reset app state using Context or Redux.
  • Catch and handle QuotaExceededError; notify users and log diagnostics.
  • Encrypt sensitive values (e.g., with crypto-js or crypto.subtle).
  • Never store raw tokens, passwords, or credentials in local storage.

üß± Architecture & Design Patterns

  • Follow MVC adapted for React:
    • Model: Services and data access
    • View: Stateless components
    • Controller: Hooks and context layers
  • Implement Model Context Protocol (MCP) to manage:
    • Session
    • Persistent memory
    • Role-based tool access
    • Application state transitions
  • Normalize API data using normalizr or reducer mapping patterns.
  • Use Progressive Enhancement and feature fallbacks.
  • Apply Policy-Based Access Control via roles, HOCs, and hooks.
  • Use Feature Flags stored in JSON or remote services.
  • Apply security design patterns: CSP, SRI, secure origin enforcement.

üß† Model Context Protocol (MCP)

  • All communication‚Äîinternal and external‚Äîmust follow MCP standards.
  • Application must support:
    • Authentication
    • Role-based access
    • Shared collaborative editing
    • Version history
  • MCP responsibilities:
    • SessionManager: Auth and session lifecycle
    • MemoryCache: Persistent local data (settings, drafts)
    • ToolRegistry: Enable tools by role
    • StateSyncEngine: Real-time updates and syncing
  • RESTful API layer:
    • Example endpoints: /session, /memory, /tools/:role, /state/:id
  • Client integration:
    • Handle session bootstrapping and live state changes via hooks or Redux

üí° Code Safety & Best Practices

  • Always use "strict mode".
  • Descriptive naming: useUserSession(), FormStateProvider, ApiService.
  • Avoid global state pollution:
    • Use modules, React.Context, Recoil, Zustand, or Redux Toolkit.
  • Validate and sanitize all user inputs:
    • Use yup, zod, or react-hook-form.
  • Log format:
    • [HH:MM:SS AM/PM] <Component>#ID: message @ line X column C
  • Provide fallbacks:
    • Use ErrorBoundary
    • Implement AbortController for timeouts
    • Retry logic with backoff

üìÑ Documentation & Versioning

  • Document:
    • All functions, custom hooks, contexts, exposed APIs
    • Use TSDoc or JSDoc
  • Maintain and change at updates:
    • README.md with setup, deployment, architecture, version, license, branding
    • CHANGELOG.md with clear version history
  • Environment:
    • .env.local for secrets
    • .env.example for safe template
  • Auto-generate /docs/:
    • API docs (Swagger/OpenAPI)
    • Data flow charts
    • Dependency graphs (madge, dependency-cruiser)
